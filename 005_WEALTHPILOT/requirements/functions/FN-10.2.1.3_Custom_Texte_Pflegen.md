# FN-10.2.1.3: Custom Texte Pflegen

> **Parent**: [CMP-10.2.1](../components/CMP-10.2.1_White_Label_Branding_Modul.md)

---

## Functional Description

Das System muss Mandanten-Admins erlauben folgende Texte anzupassen: Willkommensnachricht, Footer-Text, Impressums-URL, Datenschutz-URL, Disclaimer.

- Das System soll die Texte als Key-Value-Paare verwalten.
- Das System soll einen Markdown-Editor fuer laengere Texte bieten.
- Das System soll Default-Texte bereitstellen (falls Mandant keine eigenen setzt).

---

## Preconditions

- Admin hat die Rolle "Mandanten-Admin".
- Text-Editor ist geoeffnet.

---

## Behavior

1. System zeigt alle konfigurierbaren Texte als Formular-Felder.
2. Kurztexte (Footer, Disclaimer): Einzeiliges Textfeld.
3. Langtexte (Willkommensnachricht): Markdown-Editor mit Vorschau.
4. URLs (Impressum, Datenschutz): URL-Eingabefeld mit Validierung.
5. Admin bearbeitet die Texte und klickt "Speichern".
6. System speichert die Texte als Key-Value-Paare in der Mandanten-Konfiguration.
7. Aenderungen sind sofort in der Live-Preview sichtbar.

---

## Postconditions

- Custom Texte sind gespeichert.
- Texte werden in der Anwendung angewendet.
- Default-Texte werden als Fallback verwendet.

---

## Error Handling

- Das System soll bei ungueltigem URL-Format die Meldung "Bitte gueltige URL eingeben" anzeigen.
- Das System soll bei leerem Pflichttext den Default-Text verwenden.
- Das System soll Markdown-Inhalte sanitizen (kein HTML/Script).

---

## Acceptance Criteria (functional)

- [ ] Willkommensnachricht wird als Markdown gerendert.
- [ ] Footer-Text wird im Footer angezeigt.
- [ ] Impressums-URL fuehrt zur korrekten Seite.
- [ ] Leerer Custom-Text: Default wird verwendet.
- [ ] Markdown mit Script-Injection wird sanitized.

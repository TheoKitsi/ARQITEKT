# FN-10.1.1.2: Feature Flags Konfigurieren

> **Parent**: [CMP-10.1.1](../components/CMP-10.1.1_Mandanten_Verwaltungsmodul.md)

---

## Functional Description

Das System muss pro Mandant Module aktivieren/deaktivieren: Immobilien-Modul, KI-Advisory, Impact-Simulation, Reporting, Produktempfehlungen.

- Das System soll Feature-Flags als Toggle-Switches darstellen.
- Das System soll die Flags nach Lizenzmodell vorbelegen.
- Das System soll Aenderungen sofort wirksam machen (kein Restart noetig).

---

## Preconditions

- Nutzer hat die Rolle "Super-Admin" oder "Mandanten-Admin".
- Mandant existiert.

---

## Behavior

1. System zeigt alle verfuegbaren Module als Toggle-Switches.
2. Aktueller Zustand (aktiv/inaktiv) wird aus der Mandanten-Konfiguration geladen.
3. Admin klickt einen Toggle: System aktualisiert den Feature-Flag sofort in der Datenbank.
4. System invalidiert den Feature-Flag-Cache fuer den Mandanten.
5. Bei naechstem Request: Frontend prueft die aktualisierten Flags und blendet Module ein/aus.
6. System protokolliert die Aenderung als Audit-Event.

---

## Postconditions

- Feature-Flag ist aktualisiert.
- Module sind sofort ein-/ausgeblendet.
- Audit-Event ist protokolliert.

---

## Error Handling

- Das System soll bei Datenbankfehler den Toggle zuruecksetzen und "Aenderung fehlgeschlagen" anzeigen.
- Das System soll bei Lizenz-Einschraenkung (z.B. Starter darf kein KI-Advisory) den Toggle deaktivieren und "Nicht in Ihrem Lizenzmodell enthalten" anzeigen.

---

## Acceptance Criteria (functional)

- [ ] Toggle "Immobilien-Modul" auf ON: Modul ist sofort sichtbar.
- [ ] Toggle auf OFF: Modul ist sofort ausgeblendet.
- [ ] Starter-Lizenz: KI-Advisory Toggle ist deaktiviert.
- [ ] Aenderung wird als Audit-Event protokolliert.
- [ ] Kein Restart noetig nach Aenderung.

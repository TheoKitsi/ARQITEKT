# FN-10.2.1.2: Farb Editor

> **Parent**: [CMP-10.2.1](../components/CMP-10.2.1_White_Label_Branding_Modul.md)

---

## Functional Description

Das System muss einen Farb-Editor bieten: Primaer-, Sekundaer- und Akzentfarbe per Color-Picker. WCAG-AA-Kontrastpruefung automatisch.

- Das System soll Color-Picker-Inputs fuer 3 Farben bereitstellen.
- Das System soll automatisch den WCAG-AA-Kontrast gegen weiss/schwarz pruefen.
- Das System soll die Farben als CSS-Variablen im Mandanten-Theme speichern.

---

## Preconditions

- Admin hat die Rolle "Mandanten-Admin".
- Farb-Editor ist geoeffnet.

---

## Behavior

1. System zeigt 3 Color-Picker: Primaerfarbe, Sekundaerfarbe, Akzentfarbe.
2. System zeigt die aktuellen Farben (oder Defaults) als Vorauswahl.
3. Admin waehlt eine Farbe per Color-Picker oder Hex-Eingabe.
4. System prueft den WCAG-AA-Kontrast (min. 4.5:1 fuer Text, 3:1 fuer grosse Elemente).
5. Bei ungenuegend Kontrast: System zeigt Warnung "Kontrastverhaeltnis ungenuegend (X:1, min. 4.5:1)".
6. Admin speichert die Farben: System aktualisiert die CSS-Variablen im Mandanten-Theme.
7. Aenderungen sind sofort in der Live-Preview (FN-10.2.1.4) sichtbar.

---

## Postconditions

- Farben sind gespeichert und im Theme aktiv.
- WCAG-AA-Kontrast wurde geprueft.
- CSS-Variablen sind aktualisiert.

---

## Error Handling

- Das System soll bei ungueltigem Hex-Code die Meldung "Ungueltiger Farbcode" anzeigen.
- Das System soll bei Kontrast-Warnung das Speichern erlauben (Warnung, kein Block).

---

## Acceptance Criteria (functional)

- [ ] Primaerfarbe #1A73E8 wird korrekt gespeichert.
- [ ] Kontrastpruefung: #FFFFFF auf #1A73E8 = 4.6:1 (Pass).
- [ ] Kontrastpruefung: #FFFFFF auf #CCCCCC = 1.6:1 (Warnung).
- [ ] Aenderung ist in der Live-Preview sichtbar.
- [ ] Ungueltiger Hex-Code wird abgelehnt.

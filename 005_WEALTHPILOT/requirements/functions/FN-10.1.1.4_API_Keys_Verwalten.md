# FN-10.1.1.4: API Keys Verwalten

> **Parent**: [CMP-10.1.1](../components/CMP-10.1.1_Mandanten_Verwaltungsmodul.md)

---

## Functional Description

Das System muss pro Mandant API-Keys generieren, rotieren und widerrufen koennen. Jeder Key hat ein Erstelldatum, Ablaufdatum und optionale IP-Whitelist.

- Das System soll API-Keys als kryptographisch sichere Token generieren.
- Das System soll Key-Rotation (neuer Key + Grace-Period fuer alten) unterstuetzen.
- Das System soll eine IP-Whitelist pro Key konfigurierbar machen.

---

## Preconditions

- Mandant existiert.
- Admin hat die Rolle "Mandanten-Admin" oder "Super-Admin".

---

## Behavior

1. Admin klickt "Neuen API-Key erstellen".
2. System generiert einen kryptographisch sicheren Token (256-bit, Base64-encoded).
3. Admin konfiguriert optional: Ablaufdatum, IP-Whitelist, Beschreibung.
4. System speichert den Key-Hash (nicht den Klartext!) in der Datenbank.
5. System zeigt den Klartext-Key EINMALIG an (mit Kopier-Button und Warnung "Key wird nur jetzt angezeigt").
6. Bei Rotation: System erstellt neuen Key und setzt Grace-Period (7 Tage) fuer den alten Key.
7. Bei Widerruf: System deaktiviert den Key sofort.

---

## Postconditions

- API-Key ist generiert und gespeichert (nur Hash).
- Key wurde dem Admin einmalig angezeigt.
- Optionale Konfiguration (Ablauf, IP-Whitelist) ist gesetzt.

---

## Error Handling

- Das System soll bei Generierungsfehler einen Retry durchfuehren (max 3).
- Das System soll bei IP-Whitelist-Verletzung die API-Anfrage mit 403 ablehnen.
- Das System soll bei abgelaufenem Key die API-Anfrage mit 401 ablehnen.

---

## Acceptance Criteria (functional)

- [ ] Generierter Key ist 256-bit stark und einmalig.
- [ ] Key-Klartext wird nur einmal angezeigt.
- [ ] Rotation erstellt neuen Key mit 7-Tage-Grace fuer alten.
- [ ] Widerruf deaktiviert den Key sofort.
- [ ] IP-Whitelist blockiert Anfragen von nicht-gelisteten IPs.
